\paragraph{Module-change manifest rule.} This appendix is not a second phase table and does not release phases. It is a narrow technical ledger used only when a phase proposes new Lean modules, module moves, module merges, public API changes, import-surface changes, generator seams, or compatibility facades. Its rows are generated from implementation-scratchpad module/refactor fields; phases without module/API changes do not need meaningful rows here. The main phase table remains the primary status and release surface.
